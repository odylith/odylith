Odylith release note

What's new in v0.1.7

Odylith v0.1.7 is our first public, proof-carrying release: fail-closed managed-runtime trust, live benchmark publication, sharper grounding, and a dashboard that feels like an operator cockpit instead of scaffolding.

Odylith v0.1.7 is the first release that feels ready for public scrutiny. It combines fail-closed managed-runtime trust, benchmark proof you can inspect, sharper repo grounding, and a shell that reads like a serious delivery cockpit rather than a thin wrapper around a coding agent.

release-surface-demov0.1.6 to v0.1.7

Why it matters

This release turns benchmarking into part of the product, not a sidecar maintainer ritual. Odylith now runs a live release-proof lane across a 37-scenario corpus, measures warm and cold matched pairs, publishes family heatmaps and operating-posture graphs, and separates diagnostic tuning from proof so the public claim stays anchored to validator-backed outcomes instead of flattering prompt demos.

v0.1.7 also hardens the trust boundary. Managed runtimes now carry repo-root trust anchors outside .odylith/, verify hot-path files before odylith.cli imports, re-check the deeper runtime tree during doctor, repair, and same-version reuse, and reject insecure localhost or Sigstore-bypass release overrides outside the maintainer lane. Release, release-candidate, and test workflows now pin first-party actions, runner image, and Hatch so the supply-chain story is as explicit as the runtime story.

The grounding and operator surfaces are materially cleaner. Exact repo paths resolve directly, bootstrap and session packets compact duplicated scaffolding, host-runtime truth and registry-detail retrieval stay closer to the evidence cone, and the shell now ships a real version-story experience: an authored release spotlight, a durable release note, a built-in prompt-plus-CLI cheatsheet, and refreshed Compass, Casebook, Radar, and Registry surfaces that present Odylith as a governed system of record rather than a pile of generated pages. For elite developers and architects, that means a clearer answer to the hardest question: not just whether Odylith can act, but whether it can explain why it is safe, grounded, and worth trusting.

What changed

  • Odylith now fails closed on runtime drift with repo-root trust anchors, pre-exec hot-path verification, stricter doctor and repair checks, and a narrower release-override policy.
  • The benchmark stack now ships a 37-scenario live proof lane with warm and cold matched pairs, family-level analytics, publication-ready graphs, and reviewer docs that keep the public claim tied to validator-backed outcomes.
  • Exact-path grounding, bootstrap and session payload compaction, persistent version-story release notes, and a built-in prompt-plus-CLI cheatsheet make the product faster to trust, easier to drive, and harder to misread.
