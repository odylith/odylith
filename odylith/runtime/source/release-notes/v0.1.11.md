# Proof Without Drift

Odylith v0.1.11 turns a few important fuzzy seams into explicit product truth.
Live-proof work no longer has to rely on maintainer memory to keep one blocker
lane pinned, release planning no longer has to hide inside prose, and the
reasoning package boundary now tells the same story as the runtime itself.

Release readiness note: `v0.1.11` is not ready to publish yet. Compass now
keeps the ready brief set on maintained narrated cache, but it still has to
clear the bounded refresh bar of `<=50ms` internal hot exact reuse and `<=1s`
cold shell-safe wall clock.

- Proof-state is now a shared control plane, not a local heuristic. Delivery
  snapshots, Tribunal rows, grounded packets, shell, Compass, Registry,
  Casebook, and chatter all project the same blocker, frontier, falsification,
  and claim tier when one lane resolves cleanly, and they stay explicit about
  ambiguity when it does not.
- The benchmark now measures whether that proof discipline actually holds.
  The new `live_proof_discipline` family scores whether Odylith avoids false
  live clearance, only promotes a blocker after the hosted frontier advances
  past the previous failing phase, labels claim scope truthfully, and keeps a
  repeated fingerprint pinned to the same blocker seam instead of fragmenting
  the investigation.
- Release planning now has a governed source contract plus a real authoring
  surface. Maintainers can target workstreams at a release, inspect that truth
  through CLI and surfaces, and keep selector behavior consistent across Radar,
  Compass, and Context Engine without smuggling release intent into local
  memory.
- The runtime boundary is cleaner too. Tribunal, Remediator, and shared
  reasoning helpers now live under `src/odylith/runtime/reasoning`, while the
  benchmark harness stays under evaluation where it belongs.
