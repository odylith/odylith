# Boringly Trustworthy

Odylith v0.1.10 makes the operational path feel boring in the right way: when
you ask Compass for a real full refresh, the product either lands that deeper
state or tells you exactly that it did not.

- Explicit Compass `full` refresh now rejects deterministic local brief output,
  stale shell-safe payload reuse, and partial scoped-brief coverage for active
  workstreams. The five-minute reuse clamp stays in place, but only for
  payloads that already satisfy the full-refresh truth contract.
- The shell now projects Compass freshness honestly. A wrapper-only refresh no
  longer implies Compass is current, and failed deeper Compass rerenders stay
  visible on the Compass tab instead of hiding behind the same provider-deferred
  banner used for intentional shell-safe refresh.
- Hosted install and upgrade now end in one cleaner posture: the verified
  runtime stays live, the tracked repo pin follows it, benign verification noise
  stays quiet, and stale read-only retention trees degrade to exact remediation
  instead of false activation failure.
- Sync and release proof are quieter and more auditable. Long steps stay
  visible, large dirty-overlap writes need stronger acknowledgement, unchanged
  JSON artifacts stop pretending they were freshly regenerated, and the first
  shell refresh after install no longer flashes missing-surface failures before
  broad sync settles.
