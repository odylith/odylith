# Boringly Trustworthy

Odylith v0.1.10 makes the operational path feel boring in the right way:
Compass refresh now has one bounded contract instead of pretending there is a
second deeper mode hiding behind the same surface.

- Compass refresh now rejects stale or misleading fallback state without asking
  operators to choose between a cheap mode and a deeper mode. The bounded
  refresh path stays honest about what it reused, what it deferred, and what
  failed.
- The shell now projects Compass freshness honestly. A wrapper-only refresh no
  longer implies Compass is current, and failed Compass rerenders stay visible
  on the Compass tab instead of hiding behind the same provider-deferred banner
  used for intentional bounded refresh.
- Hosted install and upgrade now end in one cleaner posture: the verified
  runtime stays live, the tracked repo pin follows it, benign verification noise
  stays quiet, and stale read-only retention trees degrade to exact remediation
  instead of false activation failure.
- Sync and release proof are quieter and more auditable. Long steps stay
  visible, large dirty-overlap writes need stronger acknowledgement, unchanged
  JSON artifacts stop pretending they were freshly regenerated, and the first
  shell refresh after install no longer flashes missing-surface failures before
  broad sync settles.
