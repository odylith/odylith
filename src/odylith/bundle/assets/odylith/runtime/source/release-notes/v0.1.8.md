# Migration Recovery

Odylith v0.1.8 makes legacy upgrades safer, calmer, and easier to recover.

- Managed runtime trust now ignores known macOS metadata files, so `.DS_Store` and AppleDouble files do not break an otherwise healthy runtime.
- Repair and reinstall now clean up stale backup and staging residue before restaging, which makes repeated recovery attempts converge instead of compounding failures.
- Sync now normalizes legacy Radar rationale blocks before strict validation, so migrated repos can reach a working post-upgrade state faster.
- Migration also audits stale `odyssey` references, explains wrapped-runtime posture more clearly in `version` and `doctor`, and suppresses expected non-fatal verification noise when integrity checks succeed.
