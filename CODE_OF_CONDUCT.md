# Code Of Conduct

Last updated: 2026-03-27

## Our Standard

Odylith is an open-source project and collaboration space. Contributors,
maintainers, and participants are expected to keep discussion professional,
direct, and respectful.

Examples of expected behavior:

- assume good faith while still being rigorous about technical correctness
- give actionable review feedback grounded in code, behavior, or evidence
- disagree on architecture or implementation without personal attacks
- respect different levels of experience and context
- prioritize user safety, repo integrity, and clear communication

Examples of unacceptable behavior:

- harassment, threats, or discrimination
- insulting or demeaning language aimed at a person rather than the work
- repeated bad-faith disruption, trolling, or derailing
- publishing private information without permission
- knowingly misrepresenting security, release, or support status

## Scope

This code of conduct applies to:

- GitHub issues, pull requests, reviews, and commit discussions
- project email and release-related communication
- any public or private project space used to collaborate on Odylith

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code,
issues, or pull requests that violate this code of conduct.

Responses may include:

- a warning
- temporary restriction from participation
- permanent removal from project spaces

## Reporting

Report conduct issues through the repository maintainers on GitHub.

- If public reporting is acceptable, open a GitHub issue.
- If private reporting is required, use the repository owner contact options on
  GitHub rather than public discussion threads.

Reports are reviewed on a best-effort basis and handled as confidentially as
practical.
